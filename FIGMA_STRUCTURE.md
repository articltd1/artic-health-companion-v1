## Naming conventions
- Frames: `Page / <Name>`
- Components: `Component / <Type> / <Variant>`
- Instances: `Instance / <Component> / <State>`
- Sections: `Section / <Name>`

## Component structure
- Create master components for each UI element.
- Build variants for states: default, hover, focus, active, disabled, error.
- Create auto-layout frames for cards and forms.

## Responsive prototypes
- Desktop breakpoint: 1440px
- Laptop breakpoint: 1280px
- Tablet breakpoint: 768px
- Mobile breakpoint: 375px

## Notes
- Use consistent spacing and align to a 4px grid.
- Keep the sidebar and topbar visible in workspace prototypes.
- Create reusable design tokens for colors, typography, and elevation.
