# Frontend Roadmap

This roadmap defines the phased UI redesign work for ARTIC Health Companion.

## Phase 1: Design System & Foundation
- Create ARTIC brand colors, typography, spacing, and elevation.
- Build global CSS variables and base styles.
- Replace Bootstrap-only styling with design tokens.
- Create a responsive shell with sidebar + topbar.
- Document design system and component guidelines.

## Phase 2: Core Layouts
- Implement a modern landing page layout.
- Redesign login and register pages with premium onboarding.
- Create a common page shell for authenticated views.
- Add dark mode support.

## Phase 3: Dashboard & Navigation
- Build patient dashboard UI with health summary cards.
- Build doctor dashboard UI with appointment and queue cards.
- Add sidebar navigation and responsive collapse behavior.
- Add topbar with search, notifications, and user profile.

## Phase 4: Data Pages & Components
- Redesign profile page and appointments page.
- Modernize news, disease prediction, and medical record pages.
- Add reusable card, form, table, badge, and alert components.
- Add empty and loading state patterns.

## Phase 5: AI Companion Experience
- Build premium chat interface with conversation sidebar.
- Add AI quick actions and suggested questions.
- Add health cards and placeholder analytics within chat.

## Phase 6: Polish & Accessibility
- Refine spacing, contrast, and typography.
- Ensure ARIA support and keyboard navigation.
- Test responsive layouts across desktop / tablet / mobile.
- Optimize CSS, fonts, and images.

## Implementation Milestones
1. Document design system and approve colors/styles.
2. Create shared shell and baseline CSS.
3. Update patient and doctor templates with new layouts.
4. Enhance forms and component styling.
5. Add AI companion layout and minimal chat UX.
6. Final polish, accessibility, and performance fixes.

## Notes
- Do not modify backend or route behavior.
- Preserve all existing page functionality.
- Focus on UI/UX, branding, and modern product experience.
